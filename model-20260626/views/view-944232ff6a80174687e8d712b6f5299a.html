Manage implementation of improvements - Flow ()
A request for enhancements to an existing service triggers the Manage implementation of improvements process, initiated by the Continual Service Improvement Management (CSI) team. As a result, a set of actionable outputs is produced and delivered to the relevant consuming parties. In general terms, this process focuses on planning, implementing, and reviewing improvements to services and processes within the CSI capability context. It involves making decisions on which improvements to prioritize, coordinating the implementation efforts, and assessing the effectiveness of the changes made. Stakeholders typically include service managers, process owners, and other relevant parties who contribute to or are affected by the improvements. This process exists to ensure that the organization continually enhances its services and processes, delivering greater value and efficiency. Without it, the organization would struggle to adapt to changing needs and could miss opportunities for improvement, leading to stagnation. The primary output is the documented improvements delivered to consuming parties. PROCESS STEPS 1. [Prioritize improvements with stakeholders] Explanation: CSI Team and service managers use prioritization criteria to rank improvement actions. Role: CSI Team Application Service: Prioritization Matrix Application: Manual 3. [Develop implementation plan] Explanation: CSI Team creates a detailed plan for each prioritized improvement. Role: CSI Team Application Service: Project Plan Application: Project Management Tool 4. [Review plan with process owners] Explanation: CSI Manager presents the plan to process owners for approval; if not approved, the plan is revised. Role: CSI Manager Application Service: Plan Review Meeting Application: Manual 5. [Coordinate resources with service managers] Explanation: CSI Manager secures necessary resources from service managers to execute the plan. Role: CSI Manager Application Service: Resource Allocation Form Application: Manual 6. [Execute improvement tasks] Explanation: Implementation Team performs the improvement activities as per the plan. Role: Implementation Team Application Service: Task List Application: ITSM Platform 7. [Validate improvement results] Explanation: CSI Team validates that implemented improvements meet defined criteria and outcomes. Role: CSI Team Application Service: Validation Report Application: Monitoring Tool 8. [Revise and re-execute improvement] Explanation: Implementation Team reworks the improvement based on validation feedback and re-executes tasks. Role: Implementation Team Application Service: Task List Application: ITSM Platform 9. [Document implemented improvements] Explanation: CSI Team records the details and outcomes of the implemented improvements. Role: CSI Team Application Service: Improvement Record Application: Document Management 10. [Prepare and submit change requests] Explanation: CSI Manager creates "Change requests for improvements" and submits them to Change Management.
