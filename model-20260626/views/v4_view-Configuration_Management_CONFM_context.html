Configuration Management (CONFM) (context) ()
Configuration Management (CONFM) executes control objects at the Execute level, ensuring accurate and up-to-date configuration information. It progresses through the phases of maintaining configuration information and verifying configuration information, from initial data collection to final validation. It receives change information for CMDB from Change Management (CHM) to ensure that the configuration data reflects the latest changes. It produces CMDB for problem investigation for Problem Management (PM), enabling effective resolution of issues.
