Release deployment - Flow ()
The Release deployment process is triggered by a request or a scheduled event within Release & Deployment Management (RDM), leading to the successful deployment of software changes to the production environment, which are then delivered to the consuming parties. This ensures that the latest updates and features are made available to users in a timely manner. In general terms, the Release deployment process bundles changes into appropriate types of releases and effectively deploys them, ensuring that all stakeholders are informed and prepared for the updates. This process operates within the Release & Deployment Management capability, overseeing the transition of changes from development to production while making critical decisions regarding timing and resource allocation. The purpose of the Release deployment process is to provide a structured approach to deploying software changes, which adds value by minimizing disruption and ensuring that updates are delivered reliably. Without this process, the organization would face increased risks of deployment failures and inconsistencies, ultimately leading to degraded service quality for end-users. The primary output of this process is the successful deployment of changes, which is received by the consuming parties. PROCESS STEPS 1. [Receive deployment trigger] Explanation: The Release Manager initiates the process upon receiving a scheduled event or a deployment request. Role: Release Manager Application Service: Deployment Request Form Application: ITSM Platform 2. [Plan release deployment] Explanation: The Release Manager retrieves CMDB for deployment planning to assess configuration items and dependencies, and bundles changes into appropriate release types. Role: Release Manager Application Service: Release Plan Application: ITSM Platform 3. [Approve release plan] Explanation: The Change Manager reviews the release plan; if rejected the plan is returned for revision. Role: Change Manager Application Service: Approval Dashboard Application: ITSM Platform 4. [Prepare release package] Explanation: The Release Team assembles the release package including approved changes and deployment scripts. Role: Release Team Application Service: Deployment Package Application: ITSM Platform 5. [Deploy to production] Explanation: The Deployment Team executes the deployment of the release package to the production environment. Role: Deployment Team Application Service: Deployment Console Application: Deployment Tool 6. [Validate deployment success] Explanation: The Release Manager confirms deployment success via validation tests; if failed, remediation is triggered. Role: Release Manager Application Service: Validation Report Application: Monitoring Tool 7. [Remediate deployment issues] Explanation: The Deployment Team fixes deployment failures and prepares for a retry of the deployment. Role: Deployment Team Application Service: Deployment Console Application: Deployment Tool 8. [Notify stakeholders] Explanation: The Release Manager sends release deployment information for CMDB to Configuration Management, release records for PIR to Change Management, and release information for incidents to Incident Management, and informs consuming parties. Role: Release Manager Application Service: Notification Dashboard Application: ITSM Platform 9. [Conduct post-implementation review] Explanation: The Release Manager leads a review with stakeholders to evaluate deployment outcomes and capture lessons learned.
