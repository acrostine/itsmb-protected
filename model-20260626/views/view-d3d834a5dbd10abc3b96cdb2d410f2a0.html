Release planning - Flow ()
Release planning is initiated by a request or a scheduled event within Release & Deployment Management (RDM), resulting in the creation of a structured plan that outlines the upcoming changes and their deployment strategy. This ensures that all stakeholders are aligned on the timeline and resources required for the release. In general terms, the Release planning process encompasses the assessment and organization of changes into appropriate types of releases, ensuring that they are effectively bundled for deployment. It involves making critical decisions regarding the prioritization of changes, resource allocation, and timelines, while engaging various stakeholders to ensure that their needs and concerns are addressed. This process is essential within the Release & Deployment Management capability, as it sets the foundation for successful implementation of changes. The purpose of the Release planning process is to deliver structured and well-coordinated releases that minimize disruption and maximize value for the organization. Without this process, the organization would face increased risks of deployment failures, misalignment among stakeholders, and inefficient use of resources. The primary output of this process is the release plan, which is delivered to the consuming parties involved in the deployment. PROCESS STEPS 1. [Assess and categorize changes] Explanation: Evaluate approved changes for releases and change schedule to categorize them into appropriate release types; if a change is unsuitable for this release it is deferred. Role: Release Manager Application Service: Change Categorization Dashboard Application: ITSM Platform 3. [Prioritize changes for release] Explanation: Assign priority to each change based on business value, urgency, and stakeholder input to determine bundling and sequencing. Role: Release Manager Application Service: Prioritization Matrix Application: ITSM Platform 4. [Allocate resources and set timelines] Explanation: Determine resource requirements and deployment timelines, engaging IT Operations and business stakeholders to align on availability. Role: Release Manager Application Service: Resource Allocation Form Application: ITSM Platform 5. [Draft release plan] Explanation: Compile a structured release plan detailing the deployment strategy, timeline, and resource assignments. Role: Release Manager Application Service: Release Plan Document Application: Document Management 6. [Review plan with stakeholders] Explanation: Present the release plan to stakeholders for feedback; if the plan is rejected it is returned for revision. Role: Release Manager Application Service: Stakeholder Review Meeting Application: Manual 7. [Obtain final sign-off] Explanation: Submit the release plan for final approval; if rejected the plan is sent back to stakeholder review for revision and resubmission. Role: Change Manager Application Service: Approval Dashboard Application: ITSM Platform 8. [Communicate release plan] Explanation: Distribute the approved release plan to consuming parties involved in deployment.
